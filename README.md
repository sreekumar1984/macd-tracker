# F&O MACD Momentum Tracker & AI Optimizer

An enterprise-grade, high-performance daemon and analysis dashboard designed to monitor and alert on MACD crossovers, RSI anomalies, and volume metrics across all NSE F&O (Futures & Options) symbols. The app features an intelligent **Adaptive AI Optimizer** that dynamically learns from historical EOD retrospect outcomes to suppress low-conviction/false-positive signals.

---

## ⚡ Key Features

- **Multi-Timeframe MACD & RSI Tracking:** Monitors 15-minute, 45-minute, and Daily MACD lines, signals, and histograms alongside RSI levels.
- **Live F&O Bhavcopy Aggregation:** Scrapes daily Option Chain Put-Call Ratio (PCR) and Futures Open Interest (OI) changes.
- **Adaptive AI Optimizer:** Dynamically monitors daily trade performance (SUCCESS/FAILED/NEUTRAL) and tunes parameter thresholds (RSI limits, Volume thresholds, PCR ranges) to filter out whipsaws.
- **Live Interactive Dashboard:** Served locally on `http://localhost:8080` featuring:
  - Real-time stock snapshot tables.
  - Multi-column interactive regex-like filters (e.g. `>50`, `<0.7`, `TRENT`).
  - Toggleable columns with visibility memory persistence (`localStorage`).
  - System logs viewer and force-fetch status updates.
- **Automatic Housekeeping:** Self-maintains database size by purging metrics older than 30 days automatically.

---

## 🏗️ Architecture & Stack

- **Backend:** Python 3.8+ (Daemon thread-pool execution, standard `HTTPServer`).
- **Database:** SQLite (with WAL journaling enabled for concurrent read/write optimization).
- **Frontend:** Vanilla HTML5 / JavaScript (Modern UI design with smooth dark glassmorphism, local storage configurations, and asynchronous polling updates).
- **Libraries:** `pandas` (heavy-duty analytics), `yfinance` (market data queries), `requests` (API access), `sqlite3` (persistence).

## 🚀 Local Setup & Run

1. **Clone & Navigate:**
   ```bash
   git clone https://github.com/sreekumar1984/macd-tracker.git
   cd macd-tracker
   ```

2. **Initialize Environment & Start Server:**
   The easiest way is to run the automated runner shell script:
   ```bash
   bash run.sh
   ```
   *This script automatically creates a virtual environment `venv`, upgrades pip, installs `requirements.txt` dependencies, and starts the tracking daemon.*

3. **Manual Run Commands:**
   If you prefer manual control:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   python tracker.py
   ```

4. **Access the Dashboard:**
   Open your browser and navigate to:
   ```
   http://localhost:8080/
   ```

Configure your tracking parameters dynamically:
```json

  "database_path": "db/macd_history.db",

  "logging_enabled": true
```

*Note: You can also adjust these settings directly from the Web UI dashboard.*

## 🌐 Public Tunneling (Using `tunnelto`)

To share your dashboard or view it from a mobile device without port forwarding, you can use `tunnelto` or `ngrok` to expose port `8080`:

1. **Verify your local server is running on port 8080** (you should see log outputs in your terminal).
2. Start the tunnel:
   ```bash
   tunnelto --port 8080
   ```
3. Use the public URL generated by `tunnelto` to access your live dashboard from anywhere.

## 🔍 Troubleshooting

### 1. Dashboard displays "No snapshot data in database" or empty tables
- **Wait for startup sync:** When the daemon starts, it runs a background poll to retrieve data. Refresh the page after 10-15 seconds.
- **Reset Browser LocalStorage:** If you set strict filters previously, they may hide all rows automatically on load. Open your browser console (F12) and run:
  ```javascript
  localStorage.clear(); location.reload();
  ```
- **Access through Server URL:** Avoid opening `alerts_dashboard.html` as a direct local file URL (`file://...`). Browser security policies block AJAX requests on the `file://` protocol. Always use `http://localhost:8080`.

### 2. Address Already in Use (Port 8080 Error)
If another application (like a local web server or proxy) is already running on port 8080, kill the process or change the port in `tracker.py` (`run_web_server()`):
```bash
# Check what is running on port 8080
lsof -i :8080

# Kill process (replace PID with the process ID found above)
kill -9 <PID>
```
